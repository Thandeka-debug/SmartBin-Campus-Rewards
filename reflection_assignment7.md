# Reflection - Assignment 7

## Challenges

### 1. Choosing the Right Template

I compared 4 templates. Team Planning was best because it already had columns that match my work from Assignment 6 (Backlog, Ready, In Progress, In Review, Done).

### 2. Adding New Columns Without Breaking Workflow

I added Blocked and Testing columns. I placed Blocked after In Progress and Testing after In Review to maintain logical flow.

### 3. Linking Existing Issues to the Board

I had 16 existing issues from Assignment 6 (US-001 to US-016). I linked each one to the board by adding them to the Backlog column.

### 4. Setting WIP Limits for a Solo Developer

I set In Progress limit to 3 tasks, In Review to 2 tasks, and Testing to 2 tasks. This prevents context switching.

## Comparison to Other Tools

| Tool | Strength | Weakness |
|------|----------|----------|
| GitHub Projects | Integrated with code, free | Limited reporting |
| Trello | Simple, visual | No code integration |
| Jira | Powerful | Too complex, expensive |

## Lesson Learned

Kanban boards help visualize work and prevent bottlenecks, even for solo projects. Using existing issues from previous assignments saves time and maintains continuity.
