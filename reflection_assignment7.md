# Reflection: GitHub Project Templates and Kanban Board Implementation

## Challenges in Selecting and Customizing the Template

### Challenge 1: Choosing the Right Template

**The Issue:** GitHub offers four templates. I had to determine which best suited the SmartBin system.

**How I Addressed It:** I compared each template against my project needs and chose Automated Kanban for its automation features.

### Challenge 2: Adding Custom Columns Without Breaking Workflow

**The Issue:** Adding new columns could disrupt built-in automation.

**How I Addressed It:** I added "Testing" between In Progress and Done, and "Blocked" before Todo.

### Challenge 3: Defining Meaningful WIP Limits

**The Issue:** As a solo developer, limits felt artificial.

**How I Addressed It:** I set limits based on realistic solo development capacity (3 tasks In Progress, 2 tasks Testing).

## Comparison to Other Tools

| Tool | Strengths | Weaknesses |
|------|-----------|------------|
| GitHub Projects | Integrated with code, free for public repos | Limited reporting |
| Trello | Simple, easy to learn | No built-in code integration |
| Jira | Powerful reporting, industry standard | Complex, expensive |

## Lessons Learned

1. Templates save time compared to building from scratch
2. Automation reduces manual work
3. WIP limits prevent overload even for solo work
4. Visualization reveals bottlenecks
5. GitHub Projects is sufficient for academic work
