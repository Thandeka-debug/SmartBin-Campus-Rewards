# Stakeholder Analysis: SmartBin System

## Overview
This document identifies and analyzes the key stakeholders for the SmartBin Campus Sustainability Rewards System.

## Stakeholder Table

| Stakeholder | Role | Key Concerns | Pain Points | Success Metrics |
|-------------|------|--------------|-------------|-----------------|
| **Student** | Primary user who recycles items to earn points | - Easy way to track points<br>- Simple redemption process<br>- Fair point values for items | - Forgetting to bring student ID<br>- Not knowing current point balance<br- Slow redemption process | - 50% increase in student recycling participation<br>- Average user checks points 2x per week |
| **Facilities Admin** | Monitors bin status and manages maintenance | - Real-time fill level alerts<br>- Predictive maintenance schedules<br>- Reducing overflow incidents | - Bins overflowing before scheduled checks<br- No data on usage patterns<br>- Manual inspection of all bins | - 80% reduction in overflow incidents<br>- 30% reduction in collection trips |
| **Sustainability Officer** | Tracks campus recycling metrics and reports to university leadership | - Accurate recycling data<br>- Participation rates by dorm/college<br>- Environmental impact metrics (CO2 saved) | - Manual data collection from multiple sources<br>- No proof of student engagement<br>- Difficulty reporting progress | - Quarterly sustainability reports generated automatically<br>- 25% increase in recycling volume |
| **IT Administrator** | Manages system infrastructure, security, and integrations | - System uptime and reliability<br>- Data security and backups<br>- Integration with campus ID system | - Multiple systems to monitor<br>- Security vulnerabilities in new software<br>- Unplanned downtime | - 99.9% system uptime<br>- Zero security breaches<br>- Automated backups working daily |
| **University Finance Office** | Manages reward budget and tracks program costs | - Controlling reward redemption costs<br>- Budget forecasting<br>- Preventing fraud/abuse | - Students gaming the system for points<br>- Unexpected spike in reward costs<br>- Lack of spending reports | - Monthly cost reports generated automatically<br>- Fraud attempts detected within 1 hour |
| **Dining Services Manager** | Provides reward vouchers (coffee, meals) as redemption options | - Easy redemption verification<br>- Managing voucher inventory<br>- Tracking which rewards are popular | - Students showing fake redemption screenshots<br>- Running out of popular rewards<br>- Manual voucher tracking | - Digital redemption with QR codes<br>- 100% accurate inventory tracking |
| **University Marketing** | Promotes sustainability initiatives to students and donors | - Success stories for newsletters<br>- Engagement data for grant applications<br>- Positive campus image | - Lack of photos/evidence of usage<br>- No way to track campaign effectiveness<br>- Difficulty showing ROI to donors | - 5+ success stories per semester<br>- Grant applications include real usage data |
| **Developer (Thandeka Cacambile)** | Builds and maintains the system | - Clear requirements<br>- Modular code for easy updates<br>- Good documentation | - Changing requirements mid development<br>- Hard to reproduce bugs<br>- Technical debt | - 80% test coverage<br>- API documentation always up-to-date |
